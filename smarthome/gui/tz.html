Техническое Задание: GUI "Умный Дом"

Описание функциональных требований к клиентскому интерфейсу (GUI) для управления системой "Умный Дом" на платформе Aether.

1. 🏠 Общая Концепция

GUI (Графический Интерфейс Пользователя) является "Спицей" (Spoke) в архитектуре "Хаб и Спицы" (Hub-and-Spoke). Он представляет собой веб-страницу, которая подключается к центральному "Хабу" (SmartHomeService) через библиотеку Aether Cloud Client.

Интерфейс должен предоставлять пользователю возможность просматривать все подключенные устройства, управлять ими и одобрять (сопрягать) новое оборудование.

1.1. Ключевые Сущности

  • Устройство (Device): Логическая сущность, известная сервису (например, "Лампа в спальне" или "Датчик температуры на кухне").
  • Исполнитель (Actor): Тип Устройства, которым можно управлять (реле, диммер).
  • Сенсор (Sensor): Тип Устройства, которое только сообщает данные (датчик температуры, влажности).
  • Ожидающее Сопряжение (Pending Pairing): Новый Коммутатор, который подключился к сервису, но еще не одобрен пользователем.

2. 🖥️ Функциональные Экраны и Компоненты

Интерфейс должен состоять как минимум из двух основных разделов: "Панель Устройств" и "Управление Сопряжением".

2.1. Экран "Панель Устройств" (Главный экран)

Этот экран отображает все устройства, которые уже были сопряжены с системой.

Логика Загрузки: При открытии этого экрана, GUI должен вызвать метод SmartHomeServiceApi.getAllDevices(), чтобы получить полный список устройств Device[].

Компонент: Карточка Сенсора (Sensor)

Визуальный блок для каждого Device с deviceType: "SENSOR".

  • Отображает: name (например, "Датчик в гостиной").
  • Отображает: lastState + unit (например, "23.5 °C").
  • Отображает: lastUpdated (время последнего обновления).
  • Поведение: Только для чтения.

Компонент: Карточка Исполнителя (Actor)

Визуальный блок для каждого Device с deviceType: "ACTOR".

  • Отображает: name (например, "Свет на кухне").
  • Отображает: lastState (например, "ON" / "OFF" / "75%").
  • Элементы управления: Должен содержать интерактивные элементы (кнопка, переключатель, слайдер).
  • Действие: При взаимодействии с элементом управления (например, нажатии кнопки "ВКЛ"), GUI должен вызвать метод SmartHomeServiceApi.executeActorCommand(actorId, command).
📡 Требование к PUSH-Обновлению (Состояние):
GUI должен реализовывать SmartHomeClientApi. При получении PUSH-события deviceStateUpdated(device), GUI должен найти соответствующую карточку устройства на этом экране и немедленно обновить ее поля lastState и lastUpdated без перезагрузки страницы.

3. ⚙️ Экран "Управление Сопряжением" (Настройки)

Этот экран позволяет пользователю одобрять новые Коммутаторы, которые пытаются подключиться к системе.

Логика Загрузки: При открытии этого экрана, GUI должен вызвать метод SmartHomeServiceApi.getPendingPairings(), чтобы получить список PendingPairing[].

Компонент: Запрос на Сопряжение

Визуальный блок для каждого PendingPairing.

  • Отображает: commutatorId (UUID коммутатора).
  • Отображает: Список devices (например, "Обнаружено: 1 Датчик (SENSOR), 2 Исполнителя (ACTOR)").
  • Элементы управления: Кнопка "Одобрить".
  • Действие: При нажатии кнопки "Одобрить", GUI должен вызвать метод SmartHomeServiceApi.approvePairing(commutatorUuid). После успешного вызова, этот запрос должен исчезнуть из списка.
📡 Требование к PUSH-Обновлению (Сопряжение):
GUI должен реализовывать SmartHomeClientApi. При получении PUSH-события pairingRequested(pairingInfo), GUI должен немедленно добавить новый "Запрос на Сопряжение" в этот список, а также показать пользователю уведомление (например, индикатор "!" на иконке Настроек).

4. 📡 Требования к Подключению и API

4.1. Инициализация Клиента

  • GUI должен инициализировать AetherCloudClient, используя предоставленную библиотеку.
  • При запуске клиент должен подключиться к SmartHomeService (Хабу).
  • Сразу после подключения клиент должен вызвать SmartHomeServiceApi.register(), указав type: ClientType.GUI_CLIENT.

4.2. Таблица Взаимодействия с API

Краткая сводка по всем взаимодействиям GUI с API, определенным в smarthome.adsl.yaml.
