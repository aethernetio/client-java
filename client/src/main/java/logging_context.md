# Механика контекстного логирования (Log & LNode)

Система логирования AetherNet поддерживает два взаимодополняющих режима работы: явный (через экземпляры `LNode`) и амбиентный (через статический стек `Log`).

## 1. Явное логирование (Explicit Instance)
Если в коде доступен объект `LNode`, следует использовать его напрямую. Это исключает накладные расходы на работу с `ThreadLocal` стеком.

* **Применение**: Одиночные записи в асинхронных коллбэках или методах, где узел передан как аргумент.
* **Метод**: `node.info("Message")` или `node.error("Error", throwable)`.
* **Результат**: Новое сообщение создается с `parent = node`.

## 2. Амбиентное логирование (Ambient Context)
Используется, когда нужно привязать логи к контексту «прозрачно» для вызываемого кода.

* **Применение**: Оборачивание логических блоков с глубокой вложенностью вызовов.
* **Метод**: `try (var c = node.context()) { ... }`.
* **Механика**: `node.context()` вызывает `Log.push(node)`, помещая узел на вершину `ThreadLocal` стека текущего потока. Статические методы `Log.info()` автоматически вызывают `Log.get()`, подхватывая этот узел как родителя.

## 3. Критические ограничения
* **Четность аргументов**: Статические методы `Log.info(msg, data...)` и методы `node.add(data...)` принимают массив `Object...` как пары «Ключ, Значение». Передача одиночного объекта (например, `LNode`) вне пары приведет к `IllegalArgumentException`.
* **Изоляция потоков**: Стек в `Log` является локальным для потока (`ThreadLocal`). При переходе в другой поток (коллбэки Netty, асинхронные задачи) контекст должен быть либо активирован явно через `node.info()`, либо проброшен через `Log.wrap()`.
